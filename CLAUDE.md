# Instructure UX Writing Skills

This repository contains UX writing skills and resources for the Instructure design team.

## What's here
- `references/style-guide.md` — Instructure's official UX writing style guide
- `skills/` — reusable skills for checking and generating copy

## How to use
Before completing any UX writing task, always read `references/style-guide.md` first.
Apply Instructure's voice, tone, and mechanics rules to everything you write or review.

## Ground rules
- Always use Instructure's style guide as the source of truth
- When in doubt, prefer simpler, clearer language
- Never use semicolons
- Always use the Oxford comma
- Default to active voice unless avoiding user blame
